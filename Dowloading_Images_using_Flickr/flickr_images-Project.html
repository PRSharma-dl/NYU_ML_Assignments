Downloading Images using the Flickr API¶

For many image processing problems, one needs to create a dataset of images of particular classes. The ImageNet site has an enormous set of images of 1000 different classes with bounding boxes. But, this set may not have the exact image classes you are looking for. An alternative simple way to generate image datasets is via the Flickr API, which gives you access to millions of images in the Flickr database.

In this demo, you will learn to:

  • Load images from the Flickr API
  • Resize images to a desired shape
  • Save images to a file

Installing and Loading the Flickr API package¶

The Flickr API provides a simple python interface. You will need to install it via

pip install flickrapi

You can validate the installation by import the package.

In [2]:
import flickrapi
import urllib.request

To use the Flickr API, you need to apply for keys. You can do this on the Flickr website. Once you have the keys, they can be set as follows.

Loading the Images¶

The flickr.walk() function provides a generator of URLs for photos with a particular keyword. To illustrate the method, we will get a number of elephant images.

In [11]:
keyword = 'car'
dir_train = 'train'
dir_test = 'test'

class_Train = dir_train+'/'+keyword
class_Test =  dir_test+'/'+keyword

photos = flickr.walk(text=keyword, tag_mode='all', tags=keyword,extras='url_c',\
                     sort='relevance',per_page=100)

Create a directory with the name of the keyword for the images.

Next, we create a routine for displaying images.

In [13]:
# Display the image
def disp_image(im):
    if (len(im.shape) == 2):
        # Gray scale image
        plt.imshow(im, cmap='gray')    
    else:
        # Color image.  
        im1 = (im-np.min(im))/(np.max(im)-np.min(im))*255
        im1 = im1.astype(np.uint8)
        plt.imshow(im1)    
        
    # Remove axis ticks
    plt.xticks([])
    plt.yticks([])

Now, we walk through the images and save the files.

In [14]:
import warnings
    
nimage = 1300
i = 0
nrow = 64
ncol = 64
for photo in photos:
    url=photo.get('url_c')
    if not (url is None):
        
        # Create a file from the URL
        # This may only work in Python3
        response = requests.get(url)
        file = BytesIO(response.content)
        
        # Read image from file
        im = skimage.io.imread(file)
        
        # Resize images
        im1 = skimage.transform.resize(im,(nrow,ncol),mode='constant')
                
        # Convert to uint8, suppress the warning about the precision loss
        with warnings.catch_warnings():
            warnings.simplefilter("ignore")
            im2 = skimage.img_as_ubyte(im1)
    
        # Save the image
        if(i < 1000):
            local_name = '{0:s}/{1:s}_{2:04d}.jpg'.format(class_Train,keyword, i)  
        else:
            j = i%1000
            local_name = '{0:s}/{1:s}_{2:04d}.jpg'.format(class_Test,keyword, j) 
            
        skimage.io.imsave(local_name, im2)      
        print(local_name)
        i = i + 1        
    if (i >= nimage):        
        break
